    variables

    themy

    $themy: (
	'_selector': '&[data-theme][data-theme="{$}"], [data-theme="{$}"] &, [data-theme] [data-theme="{$}"] &'
) !default;

    Description

    Themy configuration variable. Properties that start with "_" are used as settings, and the other properties represent themes.

    Type

    Map

    Map structure

    Map key NameMap key DescriptionMap key TypeMap key Value
    _default

    Sets default theme, which is used to stylize the selector if no theme is specified and to get value ​​if the style does not exist in another theme

    		String or map— none
    _selector

    Sets theme selector ( will be replaced with theme name)

    		String— none

    Example

    Example configuration

    $themy: map-merge($themy, (
	'_default': 'light',
	'light': (
		'primary-color': dodgerblue,
		'accent-color': deeppink,
		'background-color': white
	),
	'dark': (
		'primary-color': #1f1f1f,
		'accent-color': violet,
		'background-color': #121212
	)
));

    mixins

    themy

    @mixin themy($args...) { ... }

    Description

    Mixin used to iterate throught themes.

    Parameters

    parameter Nameparameter Descriptionparameter Typeparameter Default value
    $args

    String and list arguments append to theme list you want to stylize, map arguments merge and override configuration.

    		Arglist— none

    Example

    All themes selection

    @include themy() { ... }

    One theme selection

    @include themy('dark') { ... }

    Multiple theme selection

    @include themy('light' 'dark', 'rainbow') { ... }

    Multiple theme selection with overriding themes styles

    @include themy('light', 'dark', (
 'light': (
 	'primary-color': deekskyblue
 ), 
  'dark': (
 	'primary-color': gray
 )
)) { ... }

    functions

    themy

    @function themy($args...) { ... }

    Description

    Function used to get theme styles.

    Parameters

    parameter Nameparameter Descriptionparameter Typeparameter Default value
    $args

    First string argument is wanted style name, second string is theme name (by default theme name is set to current iterated theme). List arguments overrides style name with styles list (if theme name not specified, overrided style name becomes theme name), map arguments merge and override configuration.

    		Arglist— none

    Example

    Getting iterated theme background color style

    body{
  @include themy() {
	  background-color: themy('background-color');
  } 
}

    Getting background color style from light theme

    body{
	background-color: themy('background-color', 'light');
}

    Getting overrided background color style from light theme

    body{
  background-color: themy('background-color', 'light', (
    'light': (
      'background-color': orange
    )
  ));
}

    Getting multiple styles from iterated theme

    .box{
	@include themy(){
		border: themy('border-width' 'border-style' 'border-color');
	}
}

    Getting multiple styles from light theme

    .box {
  border: themy('light', 'border-width' 'border-style' 'border-color');
  // or
  border: themy('border-width' 'border-style' 'border-color', 'light');
}
